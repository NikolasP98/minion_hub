# UI Design Governance (Hub mirror)

**Canonical skill: `../.claude/skills/ui-design-governance/SKILL.md`** (meta-repo root, one level above this repo). Read it if reachable — it has the full token/primitive tables. The iron rules below stand alone.

## Iron rules

1. **Semantic tokens only** in `.svelte`: no literal hex/rgb, no Tailwind palette utilities (`bg-slate-500`), no arbitrary sizes (`text-[16px]`), no raw shadows/durations/easings, no numeric z-index (`--layer-*` instead). Raw colors are legal only in `app.css` + `themes/*.ts`.
2. **⛔ Forbidden names (hard-fail):** `--accent`, `--accent-bg`, `--accent-rgb`, `--color-background`, `--color-bg1`, `--color-error`, `--color-primary`, `--color-primary-foreground`. Also: **text tokens never paint surfaces** — `--color-muted`/`--color-foreground` alias to TEXT colors; as backgrounds they go gray-on-gray on light themes. Surfaces = `--color-canvas/surface-1..3/overlay` only.
3. **Reuse primitives:** `Button` (never bare `<button>`), themed Select (never native `<select>`), `EmptyState`, `Spinner`/`Skeleton`, `Tooltip`, `FormField`, `StatusDot` (ui copy), `DataTable`, `PageHeader`, `SegmentedControl` (below), `DateRangeControls` (below), Zag wrappers. Hand-rolled copies are ratcheted debt.
3b. **Segmented toggles = `SegmentedControl`** (`$lib/components/ui`), root-caused 2026-07-19: any small mutually-exclusive choice (chart mode, revenue/qty, period, quick-range) is a bordered pill group whose ACTIVE option is an accent-TINTED pill + accent text (`bg-accent/14` + `--color-accent`) — NEVER a full-accent-fill `Button variant="primary"` (that's a primary-action style; shipped inconsistently across /finances). Don't hand-roll `Button primary/ghost` toggle rows.
3c. **Dashboard date controls = `DateRangeControls`** (`$lib/components/dashboard`), logic in the `date-range/` SDK (periods · ranges · url · storage). Standard section for EVERY dashboard (crm/finances/socials/reliability): inclusive from/to (pre-filled, rolling last-12-months default), quick ranges (1d/7d/30d/YTD/1y) + ⋯/right-click extended-range menu (MTD, last 2/3/6mo, All time), and a SMART period picker that disables a granularity when the span is shorter than one of its units (`periodEnabled`/`coercePeriod`; e.g. <28d disables Month, <365d disables Year) and snaps the current period down when it becomes disabled.
3d. **Date ranges are INCLUSIVE of the whole `to` day** (root-caused 2026-07-19, finance `periodWhere`): a `to` normalized to that day's midnight with a plain `< to` drops same-day records (from=to=Jun-1 returned zero despite Jun-1 sales). Use the half-open interval `col >= from AND col < (to + interval '1 day')` (or end-of-day). Applies to every date-range filter in the hub.
3e. **Money ALWAYS carries its currency symbol** (user directive 2026-07-19): every amount a user reads — KPI, table cell, chart axis, chart tooltip, DataTable footer aggregate, i18n interpolation — goes through `formatMoney`/`formatMoneyShort` (`$lib/utils/format`). A bare `toLocaleString()`/`toFixed()` on a money field is a bug ("3,466,783" reads as a count, not soles; shipped across the CRM revenue card, 4 socials pages, 11 stock surfaces, DataTable aggregates). (a) Never hand-roll a local `fmtMoney`/`money()`/`soles()` — 6 divergent copies existed with different decimals, only some with a symbol. (b) Where a page's `fmt` serves BOTH quantity and money, add a SEPARATE `fmtMoney` — blanket-formatting puts currency symbols on stock quantities. (c) **Currency is DATA, not a constant**: ad spend carries the ad account's ISO code (PEN *or* USD), invoices carry `fin_invoices.currency` — thread the row's currency, fall back to `PEN` only when genuinely unknown, and return `null` for a mixed-currency aggregate rather than mislabel it. (d) `DataTable` money columns set `money: true` so sum/avg aggregates keep the symbol. (e) ECharts money series need BOTH `tooltip.valueFormatter` AND `axisLabel.formatter` — the axis is the one everyone forgets.

4. **Theme = `data-theme` presets only.** Tailwind `dark:` variants are dead code here. Components read tokens; themes swap values.
5. **Never hand-edit `tokens.css`** — it is generated from `@minion-stack/design-tokens` `contract.json`.
6. **Layout contracts** (violations shipped 7 bugs, 2026-07-15): one scroll owner per screen — content pane owns scroll, section nav never scrolls away; the `(app)/+layout.svelte` fade-wrapper must stay `flex flex-col` (shells below rely on `flex:1` for height); page roots inside `SectionShell` need `flex-1 min-w-0` (or use `PageShell`) — bare `h-full` roots shrink to content width and trip EditableGrid into one column; `Button` slots children into an inner `inline-flex` row span consumer classes can't reach — override via scoped ancestor `:global(.x > span)` (POS sell `.card`, appearance `.theme-card`); Svelte scoped rules for component children need a real scoped ANCESTOR anchor — `.a :global(.b)` never matches a sibling (ShiftBanner `.mini-rail`).
7. **Primitive + panel contracts** (root-caused 2026-07-15, DataTable): forwarded classes on shared primitives are DEAD in plain scoped CSS — always `.scoped-ancestor :global(.class)` (+ `> span` for Button's inner row); selection controls = fixed `1rem` border-box both states (`--color-border-strong`/`--color-surface-2` unchecked, `--color-accent`+`--color-on-accent` checked, row-hover promotes border); NEVER dismiss panels via `position:fixed` backdrop (a `backdrop-filter`/`transform` ancestor becomes its containing block and shrinks it — DataTable's blurred thead broke click-outside) — use `<svelte:document onpointerdown>` + Escape or Zag wrappers.
8. **Bar-row lists** (user directive 2026-07-16): `label | value(s) | bar LAST` — numbers left of the bar, bar grows rightward on the final `1fr` track, and bars share one start x via container-owned tracks + `subgrid` rows (per-row grids stagger bars). Reference crm/+page.svelte `.funnel`/`.chmix`.
9. **On-accent prefers white** (user directive 2026-07-16): white text on the accent whenever it clears AA 4.5; black ONLY on bright accents where white can't (amber/cyan/green/orange). Dark accents are tuned to 600-scale shades so white passes — never flip a dark accent back to black text; `onAccentFor` (themes/contrast.ts) encodes the rule.
9b. **Severity/status ramp = one fixed semantic map** (user directive 2026-07-16): a status/severity LEVEL maps to ONE status token and reads the SAME hue on EVERY surface — chip, chart series, sankey node, dropdown dot, legend. Never let a chip palette drift from the chart palette (shipped: `info` chip `bg-accent`/red vs dropdown `--color-info-fg`/blue). Status tokens only (danger/warning/info/success + neutral/surface for calm steps); **never `--color-accent` for a status level** (accent = primary-action; reads as false alarm). Hub ramp: critical→danger, high/medium→warning, low→neutral, info→info, ok→success.
10. **Selection, charts, sticky** (root-caused 2026-07-16): list/nav SELECTION = accent-tinted surface (`bg-accent/10`) + `--color-accent` text — never a full accent fill with on-accent text (that's an action-button style; it drowns icons/dots and goes near-black-on-blue); canvas chart colors (ECharts itemStyle/palettes/sankey nodes) must be RESOLVED strings via `cssVar()`/`chartColors()` from `$lib/utils/chart-colors` — raw `'var(--token)'` renders gray and blanks gradient sankeys (DOM inline styles may keep `var()`); `position: sticky` headers need an opaque `--color-surface-*` background — transparent variants let content bleed through; every open-able panel has explicit dismissal (outside-pointerdown + Escape, or a Zag wrapper) — a `{#if open}` menu with no close path is a bug.
10b. **Personalized nav order applies in EVERY nav renderer** (root-caused 2026-07-19): the user's drag-reordered sidebar order lives in prefs (`navOrder`) and is applied through the shared `layout/nav-order.ts` helper (`readNavOrder` + `orderSections`/`orderItems`). Desktop `Sidebar` AND the mobile hamburger (`Topbar`) must both sort through it — iterating raw `getSections()`/`section.items` silently ships a different order (shipped: mobile menu ignored the reorder). Any new surface that lists nav sections/items sorts via `nav-order.ts`.
11. **Gates after every UI change:**
   ```bash
   bun run lint:design && bun run lint:tokens
   ```
   Changed-file debt may only DECREASE (CI-enforced). Exceptions: `scripts/design-lint-exceptions.json`, exact file+rule+numeric cap+category+reason. Runtime-authored vars: register in `scripts/token-integrity.mjs` maps.
