Purpose: Currently publishSkill() calls saveSkill() if dirty but proceeds to publish even if the save failed. This could publish stale or incorrect data. The fix re-checks the dirty flag after save and aborts if it's still true.

With:
```typescript
export async function publishSkill() {
  // Flush any pending save first
  if (skillEditorState.dirty) await saveSkill();

  // Re-check: if still dirty, save failed — abort publish
  if (skillEditorState.dirty) {
    skillEditorState.publishError = 'Cannot publish — unsaved changes could not be saved. Please try again.';
    return;
  }

  skillEditorState.publishing = true;
```

The rest of the function (lines 289-312) remains unchanged.

This works because `saveSkill()` sets `skillEditorState.dirty = false` only in the success path (line 277), and leaves it `true` if the fetch throws or returns an error. So re-checking dirty after await saveSkill() is a reliable signal of save failure.
