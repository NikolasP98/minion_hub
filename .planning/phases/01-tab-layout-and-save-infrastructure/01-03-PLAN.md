<task type="auto">
  <name>Task 1: Fix unreachable disconnect banner (PLSH-03 blocker)</name>
  <files>src/routes/settings/+page.svelte</files>
  <action>
The disconnect banner at line ~431 inside the `{:else}` block is structurally unreachable because `conn.connected` is always true in that branch (the outer `{#if !conn.connected}` on line ~399 guards the disconnected state).

Fix: Move the banner outside the `{#if !conn.connected}` / `{:else}` chain entirely. Place it as the first element inside the tab panel div, before the `{#if !conn.connected}` block. This way it renders whenever `!conn.connected && isDirty.value` is true regardless of which branch the config is in.

The fixed structure for each gateway config tab panel div should be:

```svelte
<div class="tab-panel absolute inset-0 flex flex-col overflow-hidden" ...>
  <!-- Disconnect + dirty banner (top-level — must not be inside the :else branch) -->
  {#if !conn.connected && isDirty.value}
    <div class="mx-6 mt-4 px-4 py-3 rounded-lg border border-amber-500/30 bg-amber-500/10 flex items-center gap-3 shrink-0">
      <span class="text-amber-400 text-xs font-medium">Disconnected</span>
      <span class="text-muted-foreground text-xs flex-1">
        Changes will be saved automatically when reconnected.
      </span>
    </div>
  {/if}

  {#if !conn.connected}
    <!-- "Not connected" placeholder (unchanged) -->
  {:else if configState.loading && !configState.loaded}
    <SettingsSkeleton />
  {:else if configState.loadError}
    <!-- error state (unchanged) -->
  {:else}
    <div class="flex-1 flex flex-col min-h-0 overflow-y-auto relative" bind:this={scrollContainers[tab.id]}>
      <!-- banner is no longer here — remove the old unreachable {#if} block -->
      ...content...
    </div>
  {/if}
</div>
```

Remove the now-dead `{#if !conn.connected && isDirty.value}` block from inside the `{:else}` branch. Do not change any other logic.

Note: add `shrink-0` to the banner div to prevent it from compressing when the tab panel uses flex layout.

<task type="auto">
  <name>Task 2: Remove duplicate ALL_TABS — import TABS from config-schema.ts</name>
  <files>src/lib/components/settings/SettingsTabBar.svelte</files>
  <action>
`SettingsTabBar.svelte` currently defines its own `ALL_TABS` constant that is an exact duplicate of `TABS` from `config-schema.ts`. The plan specified importing TABS but the executor used an inline copy instead.

Fix:
1. Add `import { TABS } from '$lib/utils/config-schema';` to the script block (after existing imports).
2. Remove the `const ALL_TABS: { id: string; label: string; icon: string }[] = [...]` declaration entirely.
3. Replace all references to `ALL_TABS` in the component with `TABS`. There are two: the `$derived` filter expression and the template `{#each tabs as tab}` (the latter references `tabs`, which is derived from `ALL_TABS`).

The ICON_MAP already has all required icon names (Brain, Bot, Radio, Shield, Server, Palette, HardDrive, DatabaseBackup) — no icon additions needed.

Keep the `USER_TABS` set and the `$derived` tabs filter exactly as-is, just swapping `ALL_TABS` for `TABS`:

```typescript
import { TABS } from '$lib/utils/config-schema';

// Remove: const ALL_TABS = [...]

const USER_TABS = new Set(['appearance']);

const tabs = $derived(
  isAdmin.value ? TABS : TABS.filter((t) => USER_TABS.has(t.id))
);
```

The icon lookup `ICON_MAP[tab.icon]` still works because TABS and the old ALL_TABS have identical icon string values.
