## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap in validation rows, close button padding |
| sm | 8px | Gap between validation findings, panel section gaps |
| md | 16px | Panel header/footer horizontal padding (matches existing modal: 1.25rem вүҲ 20px), section padding |
| lg | 24px | Panel default width rhythm reference (280px panel = 17.5 Г— 16px grid) |
| xl | 32px | Major layout gaps in book spread |
| 2xl | 48px | (not used in this phase) |
| 3xl | 64px | (not used in this phase) |

Source: existing page.svelte styles (modal padding: 0.875rem / 1.25rem / 0.75rem map to sm, md, sm rhythm).

Exceptions:
- Toolbar button padding: 0.375rem 0.625rem (6px Г— 10px) вҖ” existing non-4-grid exception inherited from `.toolbar-btn`
- Validation panel fixed width: 280px вҖ” not a spacing token, it is a layout dimension
- Panel header height: 2.75rem (44px) вҖ” matches toolbar height for visual alignment

## Color

All colors are CSS custom properties resolved at runtime from the active theme preset. No hardcoded hex values in component styles вҖ” always use the CSS variable with a fallback for the default "New York" theme.

| Role | CSS Variable | Fallback | Usage |
|------|--------------|----------|-------|
| Dominant (60%) | `--color-bg` | #09090b | Panel background surface |
| Secondary (30%) | `--color-bg2` | #18181b | Panel header/footer backgrounds, finding row backgrounds |
| Accent (10%) | `--color-accent` | #3b82f6 | Fix button foreground, focus rings on panel interactive elements |
| Error semantic | `--color-error` / `--color-destructive` | #ef4444 | Error finding rows, error icon, validation button error state, disabled-button tooltip indicator |
| Warning semantic | `--color-warning` | #f59e0b | Warning finding rows, warning icon, validation button warning state |
| Success semantic | `--color-success` | #22c55e | Publish success toast, all-clear validation button state |
| Muted | `--color-muted` | #a1a1aa | Chapter group sub-labels, passing count text, close button |
| Border | `--color-border` | #27272a | Panel border, finding row separators |

Source: `src/app.css` `@theme` block. Confirmed present: `--color-error` does not exist as a token (the codebase uses `--color-destructive` for the semantic token but also uses `--color-error` as a fallback-pattern alias in component styles). Use `var(--color-destructive, #ef4444)` consistently for errors.

Accent reserved for:
- Fix button text/icon foreground
- Focus ring on panel's interactive elements (close button, Fix buttons)
- Required field asterisk in condition editor (already established)

Accent is NOT used for: validation row backgrounds, error/warning indicators (those use semantic colors), panel border.

## Component Contract

### ValidationPanel

New component: `src/lib/components/builder/ValidationPanel.svelte`

**Layout:** Fixed-width right-side panel. The `.dag-page` section becomes `display: flex; flex-direction: row`. ChapterDAG takes `flex: 1; min-width: 0`. ValidationPanel has `width: 280px; flex-shrink: 0` when visible, `display: none` when `showValidation` is false.

**Panel open/close transition:** `transition: width 0.2s ease, opacity 0.15s ease`. When closed: `width: 0; opacity: 0; overflow: hidden`. This avoids layout jump without needing a JS animation library.

**Panel width persistence:** Write `showValidation` panel width to `localStorage` under key `validation-panel-width`. Default: 280px. Min: 220px. Max: 480px. (Width resize is at executor discretion вҖ” see RESEARCH.md Pitfall 2.)

### Publish Button (modified)

**Disabled state contract:**

```css
.toolbar-btn:disabled {
  opacity: 0.5;
  cursor: not-allowed;
}
```

- `disabled` attribute set when `skillEditorDerived.validationCounts.errors > 0 || skillEditorState.publishing`
- `title` attribute: `"Fix N error(s) before publishing"` when errors > 0 (N = actual count, pluralized)
- `title` attribute when no errors: existing strings ("Publish to shared space" / "Republish with latest changes")

**onclick handler:** When errors = 0 and warnings > 0: set `skillEditorState.showValidation = true`, set `skillEditorState.publishAnyway = true`, return early. When errors = 0 and warnings = 0: call `publishSkill()` directly.

### Validation Toolbar Button (unchanged visual contract, behavior update)

Existing `.validation-btn` with `.error` / `.warning` / `.ok` modifier classes вҖ” keep identical. Toggle behavior changes from modal to panel: `onclick={() => skillEditorState.showValidation = !skillEditorState.showValidation`.

### Toasts (replacing publish-error banner)

Positions and durations from existing `toast.svelte.ts`:

| Event | Function | Title | Description | Duration |
|-------|----------|-------|-------------|----------|
| Publish success | `toastSuccess()` | "Skill published!" | (none) | 4000ms (default) |
| Server/HTTP error | `toastError()` | "Publish failed" | Error message or "Could not reach server. Please try again." | 5000ms (override: `{ duration: 5000 }`) |

Toast placement: `top` (existing `toaster` configuration вҖ” do NOT change).

Three modals need `role="dialog"` moved from backdrop div to inner modal div:

## Interaction Contract

### Panel Open Triggers

1. User clicks "Validation" toolbar button вҶ’ toggle `showValidation` boolean
2. User clicks "Publish" with errors present вҶ’ button is disabled, no action fires
3. User clicks "Publish" with warnings but no errors вҶ’ `showValidation = true`, `publishAnyway = true`, publish is deferred

### Panel Close Triggers

1. User clicks "Г—" close button in panel header
2. User clicks "Validation" toolbar button again (toggle)
3. User opens a chapter editor (panel stays OPEN вҖ” chapter editor overlays on top, panel remains behind it)

Note: panel does NOT close on Escape key (unlike the old modal). ESC closes the chapter editor overlay only.

### Finding Reactivity

Findings update reactively via `$derived` from `skillEditorDerived.validationFindings`. No manual refresh needed. When chapter editor closes, findings re-derive automatically.

### "Publish Anyway" Visibility

`publishAnyway` flag is `true` only when: user attempted publish вҶ’ warnings-only path вҶ’ panel auto-opened. The flag resets to `false` when:
- Publish completes (success or error)
- User closes the panel
- User navigates away (cleanupSkillEditor)

The "Publish Anyway" button ONLY appears in the panel footer when `publishAnyway === true`.

### Disabled Button Behavior

`cursor: not-allowed` must be applied even though the button is `disabled` (pointer events are suppressed on disabled elements by default вҖ” apply cursor via CSS `:disabled` selector on the button itself, not a wrapper). The `title` tooltip renders via browser native `title` attribute вҖ” consistent with existing save indicator tooltip pattern.
